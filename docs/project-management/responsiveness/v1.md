# Mobile UI Optimization Notes

## /chat/chatid

* **Message Input Responsiveness**

  * Currently, input is not responsive when messages inside `/chat/id` are long.
  * Fix horizontal overflow to ensure the input remains usable.

* **Side Controls on Mobile**

  * Should be displayed as a block element, always visible.
  * Include chat timestamp.
  * Timestamp tooltip should appear when user clicks on the `MessageBubble` (since hover is not available).

* **MessageReactionPicker and Notification Popover**

  * Ensure these components are responsive.
  * Prevent overflow outside the screen on small devices.

## /temporary

* **Buttons**

  * `Start Temporary Session` and `Create Full Account` buttons have very small height on small screens.
  * Increase height for better accessibility and touch targets.


If the message is long and no spaces, it overflows on mobile. Forcefully cut the text if it reaches the max width.

On mobile, when I opened MessageReactionPicker, position it on the center of the screen, around the top of text input.
