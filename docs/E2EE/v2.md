# E2EE Issue Observation

## Summary

End-to-End Encryption (E2EE) is functioning correctly during active sessions, but issues occur after page refresh or when opening a conversation for the first time.

## Observed Behavior

### 1. Active Chat (Working Case)

* When the receiver already has the conversation open (`/chat/:id`):

  * Messages are successfully decrypted
  * Works for both private and group chats
  * Real-time incoming messages decrypt correctly

### 2. After Page Refresh (Issue)

* When the page is refreshed:

  * Previously decrypted messages become **blank**
  * This affects both:

    * Received messages
    * Sender’s own messages

### 3. Opening Chat After Receiving Message (Issue)

* If the receiver:

  * Does **not** have `/chat/:id` open
  * Receives a message
  * Then opens the conversation

* Result:

  * Messages appear **blank**
  * Decryption does **not** occur

### 4. Subsequent Messages While Chat is Open (Working Again)

* If the chat is already open (`/chat/:id`) and a new message arrives:

  * Decryption works again
  * Messages display correctly

## Key Insight

* Decryption only works **when the chat session is already active**
* Messages fail to decrypt when:

  * The page is refreshed
  * The chat is opened after receiving messages

## Possible Cause (Initial Hypothesis)

* Encryption keys or session data may not persist after refresh
* Decryption may rely on in-memory state instead of persistent storage
* Key initialization might only occur during active socket/session events

## Next Step

* Investigate key persistence and retrieval on page load
* Ensure decryption keys are available before rendering messages

---

## Fix Applied

### Root Cause
Messages were fetched from the server with `content: null` (E2EE encrypted) and rendered immediately before any decryption pass ran. Decryption only happened for real-time socket messages, not for messages loaded from the server or from IndexedDB after refresh.

### Solution
Added a **decryption pass** in `useMessages.ts` that:
1. Runs whenever messages change (after page load, after key exchange, after new messages arrive)
2. Finds all messages with `isE2ee: true`, cipher fields present, but `content: null`
3. Decrypts them using the conversation key from IndexedDB
4. Updates **both** the chatStore (live messages) and TanStack Query cache (server messages)
5. Tracks decrypted message IDs to avoid re-decrypting

### Files Changed
- `frontend/src/hooks/useMessages.ts` — Added `decryptMessages` effect + dual-store update
